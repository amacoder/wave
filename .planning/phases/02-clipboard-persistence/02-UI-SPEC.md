# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Clipboard Persistence.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase UI Surface Assessment

Phase 2 introduces **no new UI surfaces, components, or screens**. The entire phase is a surgical logic change inside `TextInserter.swift` — three NSPasteboard call-site edits. The only user-perceivable behavioral change is silent:

1. After dictation and paste, Cmd+V still re-pastes the transcribed text (clipboard not overwritten).
2. Clipboard managers (Maccy, Raycast, Paste) do not log the transcription in their history.

Because no new views, dialogs, popovers, or interactive elements are introduced, this UI-SPEC documents the **inherited design contract** that governs any incidental UI state changes (e.g., menu bar icon, no new feedback surfaces needed) and declares what the phase must NOT introduce.

Source: RESEARCH.md — "Phase 2 is a surgical, three-change fix to `TextInserter.swift`. Zero new files are required."
Source: REQUIREMENTS.md — CLIP-01, CLIP-02, CLIP-03 are all behavioral requirements with no visual component.

## Interaction Contract

### What Phase 2 Changes (User-Perceived)

| Before | After | Requirement |
|--------|-------|-------------|
| After dictation + paste, Cmd+V produces the old clipboard item | After dictation + paste, Cmd+V re-pastes the transcription | CLIP-01 |
| If user copies mid-window, the 0.5s restore overwrites their copy | No restore occurs; user's copy is preserved | CLIP-02 |
| Clipboard managers log the transcription as a history entry | Clipboard managers skip logging (TransientType marker) | CLIP-03 |

### What Phase 2 Must NOT Add

- No new overlay states (idle/recording/transcribing/done states are Phase 3)
- No toast, banner, or notification confirming "clipboard ready"
- No status indicator in the menu bar popover for clipboard state
- No user-facing settings toggle for clipboard persistence behavior
