The final `insertText` method body should be approximately:
```swift
func insertText(_ text: String) {
    print("TextInserter: inserting via clipboard + CGEvent Cmd+V")

    let pasteboard = NSPasteboard.general

    // Write transcription to clipboard — no save/restore (CLIP-01)
    pasteboard.clearContents()
    pasteboard.setString(text, forType: .string)

    // Mark as transient so clipboard managers skip logging (CLIP-03)
    pasteboard.setData(Data(), forType: .transientContent)

    // Snapshot for future restore guard (CLIP-02)
    let changeCountAfterWrite = pasteboard.changeCount

    // Clear any held modifier keys first
    clearModifierKeys()

    // Small delay to ensure modifiers are cleared
    usleep(50000) // 50ms

    // Simulate Cmd+V
    simulatePaste()

    // CLIP-02: changeCountAfterWrite available if restore path ever re-introduced.
    // guard pasteboard.changeCount == changeCountAfterWrite else { return }
    _ = changeCountAfterWrite
}
```

**Critical constraints:**
- Only ONE `clearContents()` call per insertText invocation (calling it twice wipes the first write).
- `setData` for TransientType must come AFTER `setString` and BEFORE `clearModifierKeys` — same pasteboard transaction.
- changeCount snapshot must come AFTER both `setString` and `setData` writes.
- Do NOT add any new UI, toast, or notification — this is a silent behavioral fix.
