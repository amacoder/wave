## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline padding on label/icon gaps |
| sm | 8px | Spacing between elements within a row (HStack spacing) |
| md | 16px | Default section padding, List row inset |
| lg | 24px | Section-to-section vertical gaps inside Form |
| xl | 32px | Tab content top/bottom padding |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- App icon in list row: 24×24pt fixed size (NSImage resized via `.frame(width: 24, height: 24)`)
- Settings window `.padding(20)` on outer TabView — matches existing tabs exactly (source: SettingsView.swift line 50)
- Exclusion List `.frame(minHeight: 200)` — minimum to show ~8 app rows before scrolling
- Settings window height expanded to 520pt when Exclusion tab is active (existing frame is 500×400; the scrollable List requires extra vertical room per RESEARCH.md Pitfall 6)

### ExclusionSettingsTab Layout

Two `Form` sections rendered via `.formStyle(.grouped)`:

**Section 1 — Automatic Suppression**
- `Toggle("Auto-suppress in fullscreen apps", isOn: $exclusionService.autoSuppressFullscreen)`
- Footer text below the toggle (inline, `.caption` weight): "Prevents hotkey activation in games and full-screen video players. Disable if you use fullscreen Xcode or Terminal."

**Section 2 — Excluded Apps**
- `List` of installed apps, each row:
  - `Image(nsImage: app.icon).resizable().frame(width: 24, height: 24)` — 24pt app icon
  - `Text(app.name)` — body weight
  - `Spacer()`
  - `Toggle("", isOn: ...).toggleStyle(.checkbox).labelsHidden()` — native macOS checkbox
- `.searchable(text: $searchText, prompt: "Search apps...")` modifier on the List
- `.frame(minHeight: 200)` on the List

**Tab item in SettingsView:**
- `Label("Exclusion", systemImage: "hand.raised")`

## States and Interactions

| State | Trigger | Visual Response |
|-------|---------|----------------|
| Loading (apps populating) | `onAppear` fires NSMetadataQuery | List shows no rows until `installedApps` publishes; no explicit spinner — system List idle state is sufficient |
| Populated | NSMetadataQueryDidFinishGathering fires | Full app list appears; search field becomes active |
| Search active | User types in search field | List filters in real time via `localizedCaseInsensitiveContains` |
| App excluded | Checkbox toggled ON | Checkbox visually checked (system blue); UserDefaults persisted immediately |
| App un-excluded | Checkbox toggled OFF | Checkbox visually unchecked; UserDefaults updated immediately |
| Auto-suppress toggle ON (default) | Default state on first launch | Toggle shows ON; footer text visible |
| Auto-suppress toggle OFF | User taps toggle | Toggle shows OFF; no other visual change |

No modal dialogs. No destructive confirmation flows. Exclusion/un-exclusion is immediately reversible.

## Window Sizing

The existing SettingsView frame is `width: 500, height: 400` (source: SettingsView.swift line 50). The Exclusion tab's scrollable List requires additional height.

**Contract:** Expand window height to 520pt when `.exclusion` tab is selected. All other tabs remain at 400pt height. Use `onChange(of: selectedTab)` in SettingsView body to update `.frame(width: 500, height: selectedTab == .exclusion ? 520 : 400)`.

This matches the guidance in RESEARCH.md Pitfall 6.

## Accessibility

|---------|----------|
